# Context

**Goal**: Daily measurable improvement of app responsiveness and documentation alignment.

**Scope**:
- Identify performance bottlenecks (CPU/network).
- Apply small, safe fixes (visibility gating, bounded concurrency).
- Audit and update `/content` documentation.

**Assumptions**:
- `unstable` branch is the target.
- Tests must pass.
- Changes must be documented.

**Definition of Done**:
- `perf/hits-YYYY-MM-DD.json` created.
- `perf/daily-perf-report-YYYY-MM-DD.md` created.
- At least one small fix or docs update (if applicable).
